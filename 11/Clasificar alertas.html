Trasfondo/Situación

En esta práctica de laboratorio clasificará las alertas de dos entornos diferentes: Windows y Linux. Usted es responsable de determinar si una alerta detectada es un positivo real o un falso positivo. Usted es responsable de clasificar si las alertas son generadas por el SIEM o por realizar escaneos de vulnerabilidades. Habrá dos entornos diferentes a los que puede aplicar las alertas generadas.

=  Acceso a Internet

Instrucciones

Parte 1: Investigar ID de Snort y CVE

Utilice sitios como mitre.org, snort.org, virustotal.com y sitios web de proveedores para buscar información sobre los siguientes ID de Snort (sid) e ID de CVE. Para cada alerta proporcione la siguiente información:

=  La lista cruzada entre ID de Snort y números de CVE cuando esté disponible.

=  La puntuación CVSS que indica la gravedad de cada uno de los siguientes ID.

=  Una breve descripción de la alerta.

Clasificará estas alertas en la Parte 2 y la Parte 3.

Pregunta:

1.  Sid 1-54630

Área de respuesta

CVE-2020-8617 – CVSS Score 7.5 High : mediante un mensaje especialmente diseñado, un atacante puede hacer que un servidor BIND alcance un estado incoherente si el atacante conoce (o adivinacon éxito) el nombre de una clave TSIG utilizada por el servidor.

2.  CVE-2021-3438

Área de respuesta

Un posible desbordamiento de búfer en los controladores de software para ciertos productos HP LaserJet e impresoras de productos Samsung podría provocar una escalada de privilegios.

3.  CVE-2020-5723

Área de respuesta

La serie UCM6200 1.0.20.22 y anteriores almacenan contraseñas de usuario sin cifrar en una base de datos SQLite. Esto podría permitir que un atacante recupere todas las contraseñas y posiblemente obtenga privilegios elevados.

4.  Sid 1-46597

Área de respuesta

Existe una vulnerabilidad de elevación de privilegios cuando el controlador DirectX Graphics Kernel (DXGKRNL) maneja incorrectamente los objetos en la memoria, también conocida como "DirectX Graphics Kernel Elevation of Privilege Vulnerability". Esto afecta a Windows Server 2016, Windows 10 y Windows 10 Servers.

5.  CVE-2020-28374

Área de respuesta

CVSS Score 8.1 High: En drivers/target/target_core_xcopy.c en el kernel de Linux anterior a 5.10.7, los atacantes remotos pueden usar una verificación de identificador insuficiente en el código de destino LIO SCSI para leer o escribir archivos a través de un recorrido del directorio en una solicitud XCOPY, también conocida como CID-2896c93811e3. Por ejemplo, un ataque puede ocurrir en una red si el atacante tiene acceso a un LUN iSCSI. El atacante obtiene el control sobre el acceso alos archivos porque las operaciones de E/S se transmiten a través de un backstore seleccionado por el atacante.

6.  Sid 1-31814

Área de respuesta

Esta actividad es indicativa de actividad de malware en un host. En este caso se detectó la señal saliente de keepalive enviada por MALWARE-CNC Win.Trojan.Darkcomet. Informes de VirusTotal: 50 proveedores de seguridad marcaron este archivo como malicioso.

7.  Sid 1-50089

Área de respuesta

CVE-2019-0885 – CVSS Score 5.9 High: Existe una vulnerabilidad de ejecución remota de código cuando Microsoft Windows OLE no valida correctamente la entrada del usuario, también conocida como 'Windows OLE Remote Code Execution Vulnerability'.

8.  Sid 1-50190

Área de respuesta

CVE-2019-3462 – CVSS Score 8.1 High: El saneamiento incorrecto del campo de redirección 302 en el método de transporte HTTP de las versiones 1.4.8 y anteriores de apt puede provocar la inyección de contenido por parte de un atacante MITM, lo que podría conducir a la ejecución remota de código en la máquina de destino. Varios servidores basados en Linux se enumeran como posibles objetivos.

9.  Sid 1-49188

Área de respuesta

Este evento se genera cuando el troyano deLinux SpeakUp intenta solicitar scripts maliciosos de los servidores C2. Informes de VirusTotal: 30 proveedores de seguridad marcaron este archivo como malicioso. Perl

10.  Sid 1-46991

Área de respuesta

CVE-2018-4243 – CVSS Score 5.9 High: Se descubrió un problema enciertos productos de Apple. iOS anterior a 11.4 se ve afectado. macOS anterior a 10.13.5 se ve afectado. tvOS anterior a 11.4 se ve afectado. watchOS anterior a 4.3.1 se ve afectado. El problema involucra el componente "Kernel". Un desbordamiento de búfer en getvolattrlist permite a los atacantes ejecutar códigoarbitrario en un contexto privilegiado a través de una aplicación diseñada.
